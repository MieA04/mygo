# MyGo 基础语法 (Basic Syntax)

MyGo 是一门现代化的静态类型编程语言，深度兼容 Go 语言生态，同时引入了大量源自 Rust、Swift 等现代语言的特性。本章将详细介绍 MyGo 的基础语法、类型系统、变量声明以及运算符。

## 1. 核心设计哲学

- **显式优于隐式**：无论是错误处理还是可见性控制，MyGo 都强调显式声明。
- **安全性**：通过空安全操作符和更严格的类型检查减少运行时错误。
- **组合优于继承**：摒弃类继承，通过 Struct 和 Trait 的组合实现多态。

---

## 2. 程序结构 (Program Structure)

每一个 MyGo 源文件都由以下部分按顺序组成：
1. **包声明 (`package`)**：定义文件所属的逻辑包。
2. **导入语句 (`import`)**：引入外部依赖。
3. **顶层声明**：包括函数、结构体、枚举、trait等。

#### 底层原理：包与导入 (Under the Hood: Packages & Imports)
*   **逻辑映射**：MyGo 直接将包映射到目录结构。同一目录下的所有文件必须声明相同的包名。
*   **编译单元**：编译器将每个包视为一个独立的编译单元。
*   **可见性**：MyGo 支持显式关键字控制可见性：
    *   `pub`：对所有包可见 (Public)。
    *   `pkg`：仅当前包内可见 (Package Private)。
    *   `pri`：仅当前文件内可见 (File Private)。
    *   **默认规则**：若未指定关键字，则遵循 Go 语言惯例：首字母大写默认为 `pub`，首字母小写默认为 `pkg`。

### 语法定义
```antlr
program: packageDecl? importStmt* (statement | annotationDecl)+ EOF;
```

## 3. 变量与常量 (Variables & Constants)

MyGo 使用 `let` 声明变量，使用 `const` 声明常量。

### 3.1 变量声明 (`let`)
变量声明支持显式类型标注和自动类型推导。

#### 语法结构
```antlr
varDecl: annotationUsage* modifier? 'let' ID (':' typeType)? ('=' expr)? ';' ;
```

当只提供类型而没有初始值时，编译器会根据类型自动赋予其**零值 (Zero Value)**：
- **数值类型 (int, float, byte, rune)**: 初始化为 `0`。
- **布尔类型 (bool)**: 初始化为 `false`。
- **字符串类型 (string)**: 初始化为空字符串 `""`。
- **结构体类型 (struct)**: 初始化为所有字段均为零值的实例（**注意：不是 `nil`**）。
- **引用类型 (*T, [], Map, chan, any, trait)**: 初始化为 `nil`。

```mygo
// 类型推断：此时编译器会推断此变量是字符串类型
let t = "string";

// 零值初始化：此时编译器知道这是一个 User 类型的变量，并赋予各字段零值
let u: User; 
```
```mygo
// 报错：只声明变量名称而无类型或初始值，编译器无法推断其空间分配。
let t; 
```

#### 完整示例
- **基础声明**：`let x: int = 10;`
- **类型推导**：`let y = 20.5;` (自动推导为 `float`)
- **延迟赋值**：`let z: string; z = "hello";`

### 3.2 元组解构声明 (Tuple Destructuring)
MyGo 支持一次性声明并赋值多个变量。此功能基于元组的自动拆装箱，在某些极端场景中会带来微小的性能损耗，但在绝大部分场景中几乎没有影响。

#### 语法结构
```antlr
modifier? 'let' '(' ID (',' ID)* ')' '=' expr ';'
```

#### 示例
```mygo
let (a, b) = (1, 2);
let (status, message) = getResult();
```

### 3.3 常量声明 (`const`)
常量在声明时必须赋值。
`const` 在不同的数据类型中有不同的常量含义。在基础数类型中，const代表着固定值不可改变；在复杂数据类型中（例如结构体），const仅仅意味着引用内容不可变，其并不限制复杂数据类型内部指向的其它内容不可变。
mygo当前并不支持完整的复杂常量。

#### 示例
```mygo
const PI = 3.1415926;
const APP_NAME: string = "MyGoApp";
```

## 4. 数据类型 (Data Types)

MyGo 的类型系统设计兼顾了底层性能和高层抽象。

### 4.1 基础数据类型

| 类型名称 | 说明 | 示例 |
| :--- | :--- | :--- |
| **int** | 平台相关的有符号整数 (通常为 64 位) | `let a: int = 100;` |
| **float** | 64 位浮点数 (等同于 Go 的 float64) | `let b: float = 3.14;` |
| **bool** | 布尔值 (`true` 或 `false`) | `let c: bool = true;` |
| **string** | UTF-8 编码的字符串 (不可变) | `let d: string = "你好";` |
| **byte** | 无符号 8 位整数 (等同于 Go 的 byte) | `let e: byte = 255;` |
| **rune** | 有符号 32 位整数 (等同于 Go 的 rune)，表示 Unicode 字符 | `let f: rune = '好';` |
| **any** | 万能类型 (等同于 Go 的 any/interface{}) | `let g: any = 123;` |
| **nil** | 空值字面量 (适用于指针、切片、Map、Channel、Any、Trait) | `let p: *User = nil;` |

#### 注意事项与用法
- **自动隐式转换限制**：MyGo 严禁基础类型间的隐式转换（如 `int` 赋给 `float`）。必须显式转换：`let f: float = 10 to float;`。唯一的例外是常量字面量的推导。
- **字符串不可变性**：`string` 一旦创建即不可修改。若需频繁操作，建议先转换为 `byte[]`。
- **nil 兼容性**：只有**引用类型**（指针、切片、Map、Channel、Any）可以被赋值为 `nil`。基础数值类型、布尔和**普通结构体变量**不能为 `nil`。
- **any 类型安全**：`any` 可存储任何值，但在调用方法或访问属性前，必须通过 `is` 进行类型检查或 `to` 强制转换。
- **数值溢出**：遵循 Go 语义，无内置溢出检查。

### 4.2 指针类型 (Pointers)

MyGo 支持指针，用于引用内存地址，但出于安全性考虑，**严禁指针运算**。

#### 示例
```mygo
let x = 10;
let p: *int = &x; // 取地址
println(*p);      // 解引用，输出 10
*p = 20;          // 修改指向的值
println(x);       // 输出 20
```

#### 注意事项
- **空指针检查**：在对指针进行解引用前，建议检查其是否为 `nil`，以避免运行时 Panic。
- **逃逸分析**：MyGo 依赖底层 Go 的逃逸分析，指针能安全地从函数返回。
- **禁止运算**：不支持类似 C 的 `p++` 操作，这确保了内存访问的边界安全。

### 4.3 复合类型 (Compound Types)

#### 切片 (Slices)
切片是动态数组，是 MyGo 中最常用的集合类型。
```mygo
let list: int[] = [1, 2, 3];
list.append(4);          // 追加元素
println(list.len());    // 长度: 4
println(list.is_empty()); // 是否为空: false
```

MyGo 为切片提供了丰富的内置方法：
- **容量与长度**：
  - `len()`: 返回切片长度。
  - `cap()`: 返回当前分配的容量。
  - `is_empty()`: 判断切片是否为空。
- **元素操作**：
  - `append(v1, v2...)`: 在末尾追加一个或多个元素。
  - `insert(index, v1, v2...)`: 在指定位置插入元素。
  - `remove(index)`: 移除并返回指定位置的元素。
  - `pop()`: 移除并返回最后一个元素。
  - `remove_range(start, end)`: 移除指定索引范围 `[start, end)` 的元素。
  - `clear()`: 清空切片（长度设为 0）。
- **查找与排序**：
  - `contains(value)`: 判断是否包含某个值。
  - `index_of(value)`: 返回值首次出现的索引，未找到返回 `-1`。
  - `sort()`: 对切片进行就地排序（需元素类型支持比较）。
  - `reverse()`: 对切片进行就地反转。
  - `max()` / `min()`: 返回切片中的最大或最小值。
- **其它**：
  - `clone()`: 返回切片的浅拷贝。

#### 映射 (Maps)
存储键值对的哈希表。
```mygo
// 使用 Map<KeyType, ValueType> 语法声明 Map 类型
// 需使用构造函数创建：Map<K, V>() 或 Map<K, V>(capacity)
let scores = Map<string, int>(); 

scores["Alice"] = 95; // 赋值
scores["Bob"] = 88;
scores["Alice"] = 100; // 重新赋值会覆盖旧值

// 读取值
let aliceScore = scores["Alice"]; // 100
```

**内置方法**：
- `keys()`: 获取所有键的切片。
- `values()`: 获取所有值的切片。
- `has(key)`: 返回布尔值，判断是否存在某个键。
- `len()`: 返回键值对数量。
- `clear()`: 清空 Map。

**迭代方式**：
```mygo
// 迭代键值对
for k, v in scores {
    println(k + ": " + v to string);
}

// 仅迭代键
for k in scores.keys() { ... } // 或者 scores.index()

// 仅迭代值
for v in scores.values() { ... } // 或者 scores.item()
```

**注意事项**：
1. **引用类型**：MyGo 中的 Map 是引用类型，必须使用 `Map<K, V>()` 构造函数显式初始化（底层对应 Go 的 `make(map[K]V)`）。
2. **零值读取**：访问不存在的键会返回对应值类型的零值。
3. **nil 安全**：未初始化的 Map 变量为 `nil`，对其进行读写会触发运行时 Panic。建议在使用前确保已分配空间。

#### 通道 (Channels)
用于协程 (`spawn`) 间的通信，体现了“通过通信来共享内存”的理念。
```mygo
let ch = chan<string>(1); // 创建缓冲区大小为 1 的通道
ch.write("hello");
let msg = ch.read();
```
- **注意**：通道操作默认是阻塞的。关闭后的通道不可再写入。

#### 元组 (Tuples)
轻量级的匿名结构。
```mygo
let (code, msg) = (200, "OK");
```
- **注意**：元组主要用于多返回值场景，不建议在复杂业务逻辑中替代命名结构体。

## 5. 运算符与优先级 (Operators & Precedence)

MyGo 的运算符设计遵循 **“显式且无歧义” (Explicit and Unambiguous)** 的核心哲学。为了提高代码的可读性并减少隐式错误，MyGo 在继承 Go 运算符体系的基础上，对自增自减、类型检查和错误处理进行了显著的改进。

### 5.1 运算符优先级 (Operator Precedence)

下表按优先级从高到低排列。同一行内的运算符具有相同的优先级，且均为**左结合**（除非另有说明）。

| 优先级 | 运算符 | 描述 | 结合性 |
| :--- | :--- | :--- | :--- |
| 7 (最高) | `( )`, `[ ]`, `.`, `{ }`, `->` | 括号、索引、成员访问、结构体实例化、方法链 | 左 |
| 6 | `!`, `&`, `*`, `++`, `--` | 一元非、取地址、解引用、**前缀/后缀**自增/自减 | 右 (一元/前缀) / 左 (后缀) |
| 5 | `?!`, `?!!`, `is`, `!is`, `to` | **错误处理解包**、类型检查、显式转换 | 左 |
| 4 | `*`, `/`, `%`, `&`, `<<`, `>>` | 乘除取模、按位与、位移 | 左 |
| 3 | `+`, `-`, `\|`, `^` | 加减、按位或、按位异或 | 左 |
| 2 | `==`, `!=`, `<`, `<=`, `>`, `>=` | 比较运算 | 左 |
| 1 | `&&`, `\|\|` | 逻辑与、逻辑或 | 左 (短路) |
| 0 (最低) | `? :` | 三元运算符 | 右 |

---

### 5.2 算术运算符 (Arithmetic Operators)

MyGo 提供标准的数学运算，并支持灵活的自增/自减表达式。

| 运算符 | 描述 | 设计考量 |
| :--- | :--- | :--- |
| `+` | 加法 / 字符串拼接 | 支持数值相加或字符串连接。 |
| `-`, `*`, `/` | 减法、乘法、除法 | 整数除法会截断小数位。 |
| `%` | 取模 (余数) | 仅适用于整数。 |
| `++`, `--` | **前缀/后缀**自增 / 自减 | **表达式行为**：支持作为表达式使用，可赋值给变量。 |

**设计深度解析：**
*   **表达式灵活性**：MyGo 允许 `++` 和 `--` 作为表达式使用，以满足开发者在特定场景下（如循环索引操作、简洁赋值）的需求。
    *   **后缀 (`i++`)**：返回变量的**旧值**，然后自增。
    *   **前缀 (`++i`)**：变量先自增，然后返回**新值**。
*   **消除歧义**：尽管允许作为表达式，但建议避免在复杂表达式中混合使用多次自增/自减（如 `x = i++ + ++i`），以保持代码清晰度。
*   **溢出行为**：遵循 Go 的底层语义，MyGo 默认不检查数值溢出。

```mygo
let i = 10;
let x = i++; // x = 10, i = 11 (后缀返回旧值)
let y = ++i; // i = 12, y = 12 (前缀返回新值)

// 在循环中简洁使用
let arr = [1, 2, 3];
let idx = 0;
while idx < arr.len() {
    println(arr[idx++] to string); // 使用后缀自增直接访问并移动索引
}
```

### 5.3 比较与类型运算符 (Comparison & Type Operators)

MyGo 强调**强类型安全性**，严禁跨类型的隐式比较和隐式转换。

| 运算符 | 描述 | 规则 |
| :--- | :--- | :--- |
| `==`, `!=` | 相等 / 不等 | 左右两边类型必须完全一致。 |
| `<`, `<=`, `>`, `>=` | 大小比较 | 仅适用于数值类型和字符串。 |
| `is`, `!is` | 类型匹配 | 检查 `any` 或 `trait` 变量的运行时实际类型。 |
| `to` | 显式类型转换 | 代替括号式转换（如 `x to float` 而非 `float(x)`）。 |

**设计深度解析：**
*   **禁止隐式提升**：在 MyGo 中，`int(10) == float(10.0)` 是非法操作。你必须显式写为 `10 to float == 10.0`。这种设计虽然略显繁琐，但能有效防止因精度丢失或意外自动转换导致的逻辑漏洞。
*   **类型安全转换 (to)**：使用 `to` 关键字使转换意图更加醒目。对于 `any` 类型，如果 `to` 转换失败，会触发运行时 Panic。
*   **模式匹配基础**：`is` 运算符不仅可以用于 `if` 语句，也是 `match` 语句进行类型分支判断的核心机制。

```mygo
let val: any = "hello";
if val is string {
    // 显式转换后调用 string 的内置方法
    println("Length: " + (val to string).len() to string);
}
```

### 5.4 错误处理解包符 (Error Handling Operators)

这是 MyGo 对错误处理模式的重大改进，旨在简化冗长的 `if err != nil` 检查。

#### `?!` (Error Propagation)
错误传播操作符，用于简化错误处理逻辑。
*   **成功**：提取并返回成功值 `T`。
*   **失败**：当前函数**立即返回**该错误（自动包装为 `Result.Err`）。
*   **语义**：等同于 Rust 中的 `?` 操作符，执行早期返回 (Early Return)。

```mygo
// 如果 openFile 失败，当前函数将直接返回错误
let f = os.Open("config.json") ?!; 
```

#### `?!!` (Panic-Unwrap)
强制解包。如果发生错误，直接触发运行时 `panic`。这通常用于程序初始化阶段或程序员确信不会失败的场景。

```mygo
// 如果正则语法错误，程序将在此处崩溃
let re = regexp.Compile(`\d+`) ?!!; 
```

---

### 5.5 指针与内存运算符 (Pointer & Memory Operators)

MyGo 通过严格限制指针操作来确保内存安全，同时保留底层性能优化的能力。

*   **`&` (取地址)**: 获取变量的内存地址。
*   **`*` (解引用)**: 访问指针指向的值。

**设计深度解析：**
*   **禁止指针算术**：MyGo 严禁 `p + 1` 或 `p[i]` 风格的指针位移。这消除了缓冲区溢出和非法内存访问的风险。
*   **自动解引用**：在访问结构体成员时，MyGo 会自动处理指针。无论 `p` 是结构体还是结构体指针，`p.field` 均可正常工作。
*   **逃逸分析**：MyGo 依赖编译器的逃逸分析，确保指针在函数返回后依然安全（即自动分配到堆上）。

---

### 5.6 逻辑与位运算符 (Logical & Bitwise Operators)

| 运算符 | 描述 | 备注 |
| :--- | :--- | :--- |
| `&&`, `\|\|` | 逻辑与 / 或 | 支持短路求值。 |
| `&`, `\|`, `^` | 按位与 / 或 / 异或 | 仅适用于整数类型。 |
| `<<`, `>>` | 左移 / 右移 | 位移量必须为无符号整数。 |

---

### 5.7 特殊运算符与复合赋值 (Special & Compound Assignment)

*   **三元运算符 `? :`**: MyGo 保留了此语法，但仅建议用于简单的条件赋值。
*   **范围运算符 `..`**: 目前主要用于 `for` 循环，表示左闭右开区间 `[start, end)`。
*   **复合赋值运算符**: `+=`, `-=`, `*=`, `/=` 等。
    *   **语义限制**：赋值语句在 MyGo 中**不是表达式**，没有返回值。这避免了类似 `if (x = y)` 的常见错误。
