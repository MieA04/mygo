# MyGo 面向对象与 Trait 系统 (OOP & Traits)

MyGo 不使用传统的 `class` 继承机制，而是采用组合 (`struct`) 和行为抽象 (`trait`) 的方式。这种设计避免了复杂的继承层级，提倡"组合优于继承"的理念。

## 1. Trait (特征/接口)

Trait 定义了一组行为（方法签名），任何实现了这些方法的类型都被认为实现了该 Trait。

### 定义 Trait

```mygo
trait Shape {
    fn Area(): float;
    fn Perimeter(): float;
}

trait Display {
    fn ToString(): string;
}
```

## 2. 实现 Trait (Methods)

MyGo 提供了灵活的机制来为类型添加方法并实现 Trait。最独特的是 `trait bind` 语法。

### 绑定式 Trait (Bind Trait)

`trait bind` 允许你将一组方法"绑定"到一个或多个类型上。这实现了逻辑的**高内聚**——将特定行为的实现代码集中在一起，而不是分散在各个结构体定义中。

```mygo
struct Circle {
    Radius: float
}

struct Rectangle {
    Width: float,
    Height: float
}

// 将 Shape 的行为绑定到 Circle 和 Rectangle
trait bind<T> (Circle | Rectangle) combs(Shape) {
    fn Area(): float {
        match this {
            is Circle => return 3.14 * this.Radius * this.Radius,
            is Rectangle => return this.Width * this.Height,
        }
    }
    
    fn Perimeter(): float {
        match this {
            is Circle => return 2.0 * 3.14 * this.Radius,
            is Rectangle => return 2.0 * (this.Width + this.Height),
        }
    }
}
```

### 方法调用

定义好绑定后，方法就像原生的一样调用。

```mygo
fn main() {
    let c = Circle{Radius: 5.0};
    fmt.Println(c.Area()); // 调用 Shape.Area 的 Circle 实现

    let r = Rectangle{Width: 3.0, Height: 4.0};
    fmt.Println(r.Area()); // 调用 Shape.Area 的 Rectangle 实现
}
```

## 3. Trait 高级特性

### 禁止与重命名 (Ban & Flip)

在复用 Trait 或组合 Trait 时，你可能不需要所有方法，或者想要屏蔽某些方法。MyGo 提供了 `ban` 指令。

```mygo
trait ReadWrite {
    fn Read();
    fn Write();
}

// 定义一个只读文件类型
struct MyReadOnlyFile { ... }

// 实现 ReadWrite，但禁止 Write 方法
trait bind (MyReadOnlyFile) combs(ReadWrite) {
    ban [Write]; // 显式禁止 Write 方法，调用会导致编译错误
    
    fn Read() { 
        // 实现读取逻辑
        fmt.Println("Reading file...");
    }
}
```

- `ban [Method1, Method2]`: 禁止特定方法。
- `flip ban [Method1]`: 反转禁止逻辑，即**只允许** Method1，禁止其他所有方法。

## 4. 泛型 (Generics)

MyGo 支持结构体、函数和 Trait 的泛型编程。

### 泛型 Trait

```mygo
trait Container<T> {
    fn Add(item: T);
    fn Get(index: int): T;
}
```

### 泛型约束 (Where Clauses)

可以使用 `where` 子句对泛型参数施加约束，要求其实现特定的 Trait。

**注意：MyGo 中 `where` 子句必须写在声明关键字之前。**

```mygo
// 定义一个泛型函数，要求 T 实现了 Shape Trait
where T: Shape
fn printArea<T>(s: T) {
    fmt.Printf("Area: %f\n", s.Area());
}

// 多个约束
where T: Shape + Display
fn printShapeInfo<T>(s: T) {
    fmt.Println(s.ToString());
    fmt.Println(s.Area());
}
```

### 泛型结构体与方法

```mygo
struct Stack<T> {
    items: []T
}

// 为泛型结构体绑定方法
trait bind<T> (Stack<T>) {
    fn Push(item: T) {
        // 假设有 append 函数
        this.items = append(this.items, item);
    }
    
    fn Pop(): T {
        // ... 实现出栈逻辑
        return this.items[0]; // 简化示例
    }
}
```
